# Deploy comemory: install script + Homebrew tap (v0.2.0 launch)

**Status:** Design — approved for plan
**Date:** 2026-06-09
**Owner:** falconiere
**Target release:** v0.2.0 (preceded by v0.2.0-rc.1 dry-run)

## Goal

Ship comemory to end users via two channels:

1. **Curl installer**: `curl --proto '=https' --tlsv1.2 -LsSf <release-url>/comemory-installer.sh | sh`
2. **Homebrew tap**: `brew install Falconiere/tap/comemory`

Pipeline must be single-tag-triggered, automatic, and produce signed-by-checksum artifacts for four targets:

- `aarch64-apple-darwin`
- `x86_64-apple-darwin`
- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`

## Out of scope

- crates.io publishing
- homebrew-core submission
- musl Linux targets
- Windows / FreeBSD / other platforms
- macOS code signing / notarization (curl-downloaded bytes lack `com.apple.quarantine`; `brew` strips it — both install paths bypass Gatekeeper unsigned-binary block in v0.2)
- Automated post-release smoke test inside a fresh GH runner

## Non-goals

- Replace the developer build-from-source flow. `scripts/install.sh` (renamed to `scripts/dev-install.sh`) stays for contributors who need `cargo install --path .`.
- Hand-maintained release workflows. cargo-dist 0.31.0 (already pinned in `Cargo.toml`) owns workflow generation; we do not patch the workflow by hand.

## Architecture

Single-tag-triggers-all pipeline. Push `vX.Y.Z` → GitHub Actions runs `release.yml` (generated by `cargo dist generate`) → gate (`scripts/check-all.sh`) → fan-out build matrix (4 targets) → upload tarballs + checksums + `dist-manifest.json` + shell installer to GitHub Releases → push updated formula to `Falconiere/homebrew-tap`.

Two repos involved:

- `Falconiere/comemory` — source + release workflow + tag
- `Falconiere/homebrew-tap` — formula auto-pushed by cargo-dist via PAT

Two end-user install paths:

- `curl --proto '=https' --tlsv1.2 -LsSf https://github.com/Falconiere/comemory/releases/latest/download/comemory-installer.sh | sh` — drops binary into `$CARGO_HOME/bin` or `~/.local/bin`, installs shell completions per `--mode` flag
- `brew install Falconiere/tap/comemory` — formula resolves prebuilt tarball from GitHub Releases per platform

No code-signing in v0.2. Document Gatekeeper caveat for users who download tarballs directly via a browser (which would tag the bytes with `com.apple.quarantine`).

## Components

### Repo: `Falconiere/comemory`

- `Cargo.toml` `[workspace.metadata.dist]` — unchanged config (cargo-dist 0.31.0, targets, tap, installers).
- `.github/workflows/release.yml` — generated by `cargo dist init` / `cargo dist generate`. Triggers on `v*` tag. Steps:
  1. Install Rust toolchain `1.95` (from `Cargo.toml` `rust-version`).
  2. Run `bash scripts/check-all.sh` as `plan` gate (fail-fast).
  3. `cargo dist build --target <T>` per matrix entry.
  4. `cargo dist host` (upload artifacts to GitHub Release).
  5. `cargo dist publish` (homebrew job pushes to tap).
- `CHANGELOG.md` — Keep-a-Changelog format. Each released version gets a `## [X.Y.Z] - YYYY-MM-DD` section; cargo-dist scrapes it for the GitHub Release body.
- `scripts/install.sh` → renamed to `scripts/dev-install.sh`. Update `justfile`, `README.md`, and `.claude/hooks/` references.
- `README.md` — install section rewritten: `brew install` first, `curl ... installer.sh | sh` second, `cargo install --git` / `dev-install.sh` for contributors.
- `docs/architecture.md`, `docs/cli-reference.md` — install pointers updated to match new README.
- New: `docs/release.md` — runbook covering Cargo.toml bump, CHANGELOG section, tag push, RC dry-run, rollback.

### Repo: `Falconiere/homebrew-tap` (new, empty)

- Created empty on GitHub. Default branch `main`. Public.
- cargo-dist writes `Formula/comemory.rb` on each release. No hand-maintained content.
- README left default (or one-liner pointing at `Falconiere/comemory`).

### GitHub secrets (in `Falconiere/comemory` repo)

- `HOMEBREW_TAP_TOKEN` — fine-grained PAT scoped to `Falconiere/homebrew-tap` with `contents: write`. Exposed to `release.yml` via env var `HOMEBREW_TAP_TOKEN` (configured in generated workflow).

### Local tooling

- `just release-dry-run` — wraps `cargo dist plan --tag vX.Y.Z-rc.1` for local validation before tag push.
- `just check` — unchanged dev gate, identical to CI gate.

## Data flow

### Cut-a-release (maintainer)

1. Bump `Cargo.toml` `version = "X.Y.Z"`.
2. Append `## [X.Y.Z] - YYYY-MM-DD` section to `CHANGELOG.md` listing changes.
3. `git commit -am "release: vX.Y.Z"` on `main`.
4. `just release-dry-run` (or `cargo dist plan`) — prints planned artifacts, no network calls.
5. `git tag vX.Y.Z && git push origin main vX.Y.Z`.

### CI (triggered by tag push)

1. `release.yml` fires on `push.tags: ['v*']`.
2. **plan-gate job** (ubuntu-latest): checkout → `bash scripts/check-all.sh`. Fail → workflow aborts, no artifacts.
3. **build matrix** (4 parallel jobs):
   - `macos-13` → `x86_64-apple-darwin`
   - `macos-14` → `aarch64-apple-darwin`
   - `ubuntu-22.04` → `x86_64-unknown-linux-gnu`
   - `ubuntu-22.04-arm` (or cross fallback on `ubuntu-22.04`) → `aarch64-unknown-linux-gnu`
   - Each: install Rust 1.95 → `cargo dist build --target <T>` → produces `comemory-X.Y.Z-<target>.tar.xz` + sha256.
4. **host job**: collect artifacts → upload to GH Release `vX.Y.Z` → attach `comemory-installer.sh` + `dist-manifest.json` + CHANGELOG section as release body.
5. **publish-homebrew job**: cargo-dist renders `Formula/comemory.rb` with per-target URLs + sha256 → clones `Falconiere/homebrew-tap` with `HOMEBREW_TAP_TOKEN` → commits formula → pushes to `main`. Skipped on pre-release tags (`-rc.N`, `-beta.N`).

### End-user install (curl)

1. `curl ... installer.sh | sh`.
2. Installer detects OS + arch via `uname -sm` → picks tarball URL → downloads → verifies sha256 against `dist-manifest.json` → extracts to `$CARGO_HOME/bin` (or `~/.local/bin`) → installs shell completions.
3. `comemory doctor` for post-install verification.

### End-user install (brew)

1. `brew tap Falconiere/tap` (once) → `brew install comemory`.
2. brew reads `Formula/comemory.rb` → picks bottle URL → downloads → verifies sha256 → links into `$(brew --prefix)/bin/comemory`.

### Dry-run (`v0.2.0-rc.1`)

Same path as above with tag `v0.2.0-rc.1`. cargo-dist treats `-rc.N` suffix as pre-release → GH Release marked pre-release → tap formula NOT updated. Validates build matrix + installer + GH upload without polluting stable tap.

## Error handling

| Failure | Detection | Recovery |
|---------|-----------|----------|
| `Cargo.toml` version ≠ tag | cargo-dist `plan` step | Delete tag (`git push --delete origin vX.Y.Z`), fix Cargo.toml, re-tag. |
| `scripts/check-all.sh` fails | plan-gate job | Workflow aborts. Fix main, delete tag, re-tag. |
| Single target build fails | build matrix job | cargo-dist `host` will not publish until all targets succeed (atomic release). Re-run failed job from Actions UI after fix. If partial artifacts uploaded, delete GH Release manually before retry. |
| `HOMEBREW_TAP_TOKEN` expired / missing scope | publish-homebrew job | GH Release for `comemory` still published (binaries reachable via curl). Rotate PAT → re-run failed job from Actions UI. cargo-dist publish job is idempotent. |
| Tap push non-fast-forward (race) | publish-homebrew git push | cargo-dist retries with rebase. Hard failure → maintainer hand-pushes formula. |
| Installer unsupported platform | `uname -sm` check | Installer exits with `error: unsupported platform '<os>/<arch>'` and lists supported targets. |
| Installer checksum mismatch | installer sha256 check | Installer aborts with `error: checksum mismatch for <file>`. No partial install on disk. |
| `$CARGO_HOME/bin` not writable | installer mkdir/write | Falls back to `~/.local/bin` per cargo-dist default; logs which directory chosen. |
| brew tarball 404 (tap restored from backup) | `brew install` HTTP error | Maintainer republishes affected version via `cargo dist host --tag vX.Y.Z`. |

All recovery steps documented in `docs/release.md`.

## Testing

### Pre-tag local validation

- `just check` — full `scripts/check-all.sh` (mirrors CI gate).
- `just test` — `cargo nextest run --all-features`.
- `just release-dry-run` — `cargo dist plan --tag vX.Y.Z` prints planned matrix and installer URL. Catches Cargo.toml/tag mismatch and missing CHANGELOG section.
- `cargo dist build --artifacts=local` — builds host-platform tarball locally; manually run installer to validate UX without GH upload.

### RC dry-run acceptance (`v0.2.0-rc.1`)

1. CI completes green end-to-end (plan-gate + 4 build jobs + host + homebrew-publish skipped due to pre-release).
2. GH Release `v0.2.0-rc.1` exists, marked pre-release, contains 4 tarballs + 4 sha256 + `comemory-installer.sh` + `dist-manifest.json`.
3. Tap repo `Formula/comemory.rb` NOT updated. If updated by mistake, reconfigure pre-release suppression and retry.
4. Curl install on local macOS arm64: `curl ... rc.1/installer.sh | sh` → binary in `~/.cargo/bin/comemory` → `comemory --version` prints `0.2.0-rc.1` → `comemory doctor` exits 0.
5. Repeat on Linux x86_64 inside `docker run -it ubuntu:22.04` with curl + bash available.

### v0.2.0 final acceptance

1. CI green; GH Release `v0.2.0` published, NOT marked pre-release.
2. `Falconiere/homebrew-tap` `Formula/comemory.rb` updated — `git log` shows commit by `github-actions[bot]`.
3. macOS arm64: `brew tap Falconiere/tap && brew install comemory` → `comemory --version` prints `0.2.0` → `comemory doctor` green.
4. Linux x86_64 (linuxbrew or curl-installer): same.
5. Uninstall path: `brew uninstall comemory` cleanly removes binary; `~/.cargo/bin/comemory` removable via `rm` after curl install.

### Regression safety net

- Existing PR CI runs `scripts/check-all.sh`. PRs that break the gate cannot merge to main, so tags from main are presumed-green before workflow gate re-runs.
- `pr-run-mode = "plan"` in `Cargo.toml` — cargo-dist comments dry-run plan on PRs that touch release-relevant files. Already configured.

### No new test code in `src/` or `tests/`

This work touches scripts, docs, workflows. The binding rule "tests in `tests/` mirroring `src/` 1:1" is unaffected. Validation is workflow runs + manual install + `comemory doctor`.

## Decision log

- **cargo-dist as orchestrator** (vs. handcrafted workflow vs. cargo-release + manual brew PR) — Cargo.toml already commits to cargo-dist 0.31.0; rebuilding what it gives free is wasted work.
- **gnu-only Linux** (vs. + musl) — defer musl to v0.3 if container users file an issue. Avoids doubling build matrix on day one.
- **No crates.io / no homebrew-core in v0.2** — narrow scope to two channels that ship immediately. Re-evaluate after v0.2 stabilizes.
- **No macOS code signing in v0.2** — both supported install paths bypass Gatekeeper quarantine. Re-evaluate if browser-download is requested.
- **CHANGELOG.md as release-body source** (vs. git-cliff vs. GitHub auto) — hand-maintained matches existing `CHANGELOG.md` (1.1K) and lets the maintainer curate user-facing notes.
- **Full `scripts/check-all.sh` gate before build** — matches CLAUDE.md "task not done until check-all exits 0" and prevents a bad tag from producing published binaries.
- **v0.2.0-rc.1 dry-run first** — full pipeline validation without polluting stable tap or hand-tagging users.
